# 文档地图同步技能

## 概述

维护 `docs/map/` 下的模块地图文件。每当 `docs/module/` 或 `docs/api/` 中有文档被创建或更新时，自动将地图中对应模块的文档链接补全或更新，保持地图与实际文档的一致性。

---

## 第一步：确认触发来源

从当前上下文判断本次触发的来源文件：

- **刚写入了哪个文档？** → 记录文件路径，如 `docs/api/SM_Organization.md`
- **属于哪个模块？** → 从文件名或对话上下文推断，如 `SM` app 的 `Organization` 模块
- **文档类型？** → `docs/module/` = 模块说明文档，`docs/api/` = API 接口文档

如无法从上下文推断，询问用户确认。

---

## 第二步：扫描地图文件

```bash
# 列出所有地图文件
ls docs/map/

# 读取地图文件内容（通常为一个或多个 .md 文件）
cat docs/map/*.md
```

**若 `docs/map/` 目录不存在或为空：**

```bash
mkdir -p docs/map
```

→ 跳转至【第四步：初始化地图】

---

## 第三步：定位并更新对应条目

在地图文件中找到与本次文档对应的模块条目，更新其链接。

**匹配规则（按优先级）：**
1. 精确匹配模块名（如 `Organization`、`User`）
2. 匹配 app 名（如 `SM`、`BDM`）
3. 匹配文件名关键词

**更新内容：**

| 文档类型 | 地图中更新的字段 |
|----------|----------------|
| `docs/module/` | 模块文档链接 |
| `docs/api/` | API 接口文档链接 |

> ⚠️ 更新格式规范请读取：`references/map-format.md`

**未找到对应条目时** → 跳转至【第五步：新增地图条目】

---

## 第四步：初始化地图（地图不存在时）

创建 `docs/map/README.md` 作为主地图文件，写入初始结构后，执行第五步新增当前模块条目。

> ⚠️ 地图初始结构请读取：`references/map-format.md`

---

## 第五步：新增地图条目（条目不存在时）

在地图文件中为当前模块新增一行条目，填入已知的文档链接，未完成的文档链接留空占位。

> ⚠️ 条目格式请读取：`references/map-format.md`

---

## 第六步：输出同步摘要

操作完成后，向用户报告：

```
✅ 文档地图已同步：docs/map/README.md

📋 变更摘要：
  - 模块：{AppName} / {ModelName}
  - 更新字段：{模块文档链接 | API文档链接}
  - 链接：{相对路径}
  - 操作类型：{新增条目 | 更新链接 | 无变化}
```
