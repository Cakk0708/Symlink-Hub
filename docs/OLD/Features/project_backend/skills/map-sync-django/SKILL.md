# 文档地图同步技能

## 概述

维护 `docs/map/` 下的模块地图文件。每当 `docs/module/` 或 `docs/api/` 中有文档被创建或更新时，自动将地图中对应模块条目的路径补全或更新，保持地图与实际文档的一致性。

> ⚠️ 地图条目格式与文件命名规则请读取：`references/map-format.md`

---

## 第一步：确认触发来源

从当前上下文判断本次触发的来源：

- **刚写入了哪个文档？** → 记录文件路径，如 `docs/api/bdm-customer.md`
- **文档类型？**
  - `docs/module/` → 对应地图中的 `模块说明:`
  - `docs/api/` → 对应地图中的 `接口说明:`
- **属于哪个 app / 模块？** → 从文件名推断，如 `bdm-customer.md` → app `BDM`，模块 `customer`

如无法从上下文推断，询问用户确认后继续。

---

## 第二步：读取地图文件

```bash
# 确认地图文件存在
ls docs/map/

# 读取地图内容
cat docs/map/README.md
```

**若 `docs/map/` 目录或 `README.md` 不存在：**

```bash
mkdir -p docs/map
```

→ 跳转至【第四步：初始化地图】

---

## 第三步：定位并更新对应条目

在地图中找到与本次文档对应的模块条目。

**匹配规则（按优先级）：**
1. 精确匹配模块路径关键词（如 `customer/`、`nodelist/`）
2. 匹配 app 区块（如 `### BDM`、`### PM`）+ 模块名
3. 匹配文件名去除前缀后的关键词（如 `bdm-customer` → `customer`）

**找到条目后，根据文档类型执行对应更新：**

| 文档类型 | 地图字段 | 操作 |
|----------|---------|------|
| `docs/module/` | `模块说明:` | 补全或替换路径 |
| `docs/api/` | `接口说明:` | 补全或替换路径 |

- 若该字段**已存在** → 替换为新路径
- 若该字段**不存在** → 在该条目下新增对应行
- 若**找不到匹配条目** → 跳转至【第五步：新增地图条目】

---

## 第四步：初始化地图（地图不存在时）

创建 `docs/map/README.md`，写入基础结构，然后执行第五步新增当前模块条目。

> ⚠️ 初始结构模板请读取：`references/map-format.md`

---

## 第五步：新增地图条目（条目不存在时）

在地图对应 app 区块末尾新增该模块条目，填入已知的文档路径。

> ⚠️ 新增条目格式请读取：`references/map-format.md`

---

## 第六步：输出同步摘要

操作完成后，向用户报告：

```
✅ 文档地图已同步：docs/map/README.md

📋 变更摘要：
  - App / 模块：{BDM} / {customer}
  - 文档类型：{模块说明 | 接口说明}
  - 路径：{modules/bdm-customer.md | api/bdm-customer.md}
  - 操作：{新增条目 | 更新链接 | 无变化}
```
