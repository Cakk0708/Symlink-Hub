# 移动端适配

## 视口配置

- **设计基准**: 375px (iPhone SE)
- **使用单位**: `rem` 或 `vw/vh` 进行响应式尺寸设置
- **目标浏览器**: iOS Safari 和 Android Chrome

## 触控交互

### 最小触控目标

- **最小尺寸**: 44x44px
- **推荐尺寸**: 48x48px

### 触控优化

- 使用 Vant 的触控优化组件
- 避免误触：增大按钮间距
- 提供触控反馈：使用 Vant 的 Toast、Dialog 等组件

## 响应式设计

### 布局策略

- 使用 Flexbox 布局
- 避免固定宽高
- 使用相对单位 (rem、vw、vh、%)

### 字体适配

```css
/* 使用 rem 单位 */
font-size: 0.14rem; /* 14px */

/* 使用 vw 单位 */
font-size: 4vw; /* 视口宽度的 4% */
```

## 设备兼容性

### iOS Safari

- 支持 iOS 12+
- 使用 `-webkit-` 前缀处理兼容性
- 处理安全区域（刘海屏）

### Android Chrome

- 支持 Android 5.0+
- 测试不同分辨率设备
- 处理虚拟键盘遮挡

## 测试

- 在真实移动设备上测试
- 使用 Chrome DevTools 设备模拟
- 测试不同屏幕尺寸
- 测试横竖屏切换

## 性能优化

- 懒加载图片和组件
- 压缩资源文件
- 使用 CDN 加速
- 减少重绘和回流
