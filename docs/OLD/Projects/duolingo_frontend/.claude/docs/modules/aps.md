# APS (订单管理) 模块说明

## 模块定位

- **路径**: `/aps`
- **别名**: 订单管理
- **所属**: 核心业务模块

## 功能概述

APS (Automatic Processing System) 模块是系统的核心业务模块，负责订单的创建、管理、执行状态跟踪等功能。订单是系统的主要业务对象，通过使用账号为客户提供自动化服务。

## 核心功能

### 1. 订单管理
- **订单列表**: 展示所有订单信息
- **多条件搜索**: 客户、账号、订单类型、状态等
- **状态管理**: 订单状态流转控制
- **批量操作**: 批量处理订单

### 2. 订单创建
- **账号选择**: 搜索并选择执行账号
- **订单类型**: EXP、GEMS、3X_XP、SIGN 四种类型
- **参数设置**: 计划数量、执行周期等
- **订单预览**: 创建前预览订单信息

### 3. 订单执行
- **状态跟踪**: 实时查看订单执行状态
- **手动控制**: 开始、暂停、完成、重置、取消
- **进度展示**: 显示执行进度和数据
- **结果记录**: 记录执行结果和日志

### 4. 订单详情
- **基本信息**: 订单号、类型、状态等
- **执行数据**: 执行过程中的详细数据
- **关联信息**: 关联的客户、账号、组织
- **时间记录**: 创建时间、开始时间、完成时间等

## 订单状态流转

### 状态定义
- **等待** (pending): 订单已创建，等待执行
- **进行中** (running): 订单正在执行
- **完成** (completed): 订单执行成功
- **取消** (cancelled): 订单被取消
- **错误** (error): 执行过程中出现错误

### 状态转换
```
等待 → 进行中 → 完成
  ↓    ↓      ↓
取消  暂停   重置
```

## 技术实现

### 路由配置
```javascript
{
  path: '/aps',
  component: Layout,
  meta: { title: '订单管理', icon: 'List' },
  children: [
    {
      path: 'order',
      name: 'OrderManagement',
      component: () => import('@/views/APS/order/index.vue'),
      meta: { title: '订单列表', icon: 'Document' }
    }
  ]
}
```

### 数据接口
- `GET /APS/orders` - 获取订单列表
- `POST /APS/orders` - 创建订单
- `GET /APS/orders/{id}` - 获取订单详情
- `PATCH /APS/orders/{id}` - 更新订单状态
- `DELETE /APS/orders/{id}` - 删除订单

## 业务逻辑

### 订单执行流程
1. **订单创建**: 选择账号、设置参数
2. **订单排队**: 进入执行队列
3. **开始执行**: 调用对应执行脚本
4. **状态监控**: 实时更新执行状态
5. **完成记录**: 记录执行结果

### 错误处理
- **网络错误**: 自动重试机制
- **执行超时**: 超时自动停止
- **账号异常**: 切换备用账号
- **系统错误**: 记录日志并通知
